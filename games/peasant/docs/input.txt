
Apple II by default has non-buffered input you have to always check.

We have a few routines for doing this.

When we moved to fully double-buffered graphics it turns out the lag was
enough that keypresses not appearing caused trouble.

As a hack to work around that the screen copy code will stop and call the
keyboard handler and buffer up keypresses which are then handled after
it is done.  This allows OK typing but has issues as all users of this
routine have to add code to drain the buffer or else things get stuck.

A better solution might be to sprinkle calls to read keyboard around
and if pressed drop everything and instantly write to both graphics pages
the key that was pressed.  No interrupts on Apple II keyboard so this
is trickier.
